# Highlight（浏览器扩展）

在 **任意常见网页**（`http` / `https`，见下）上**划选即高亮**，并可选 **精读模式**（压暗页面、突出已高亮内容）。高亮按**页面 URL**（忽略 hash）持久化到本地，刷新后自动尝试恢复。

基于 **Chrome Manifest V3**，适用于 Chromium 系浏览器（Chrome、Edge 等）。

---

## 功能概览

| 能力 | 说明 |
|------|------|
| **启用高亮** | 开启后，在页面中划选文本松手即可用 `<mark>` 包裹并着色。 |
| **高亮颜色** | 弹窗内 6 种颜色（黄 / 绿 / 蓝 / 粉 / 橙 / 紫）；**仅影响之后新划选**，已保存的高亮保留创建时的颜色。 |
| **精读模式** | 开启后压暗非高亮区域，高亮与扩展自带浮层保持清晰。 |
| **本页高亮面板** | 页面右下角浮层：列表展示当前页高亮摘要，点击可滚动定位并短暂闪烁提示；可折叠、拖动、关闭（关闭后可用「高亮」小按钮重新打开）。 |

---

## 安装（开发者模式）

1. 克隆或下载本仓库到本地。
2. 打开 Chrome：**扩展程序** → 打开**开发者模式**。
3. 点击 **加载已解压的扩展程序**，选择项目根目录（含 `manifest.json` 的文件夹）。

---

## 使用说明

1. 打开任意支持的网页（见下文「注入范围」）。
2. 点击工具栏中的扩展图标，打开弹窗。
3. 打开 **启用高亮**，按需选择 **高亮颜色**、**精读模式**。
4. 在正文里划选文字即可添加高亮；通过右下角面板快速跳转。

**注意：**

- 高亮列表、精读等逻辑会忽略扩展注入的**面板内部**选区，避免误触。
- 恢复高亮依赖存储的文本片段在 DOM 中仍可匹配；动态站点若大幅改写正文，可能出现无法完全恢复的情况。

## 注入范围（`content_scripts.matches`）

当前为 **`<all_urls>`**：在浏览器允许注入的页面上都会加载脚本与样式（涵盖绝大多数 `http` / `https` 站点）。

**通常不会注入**的内置页示例：`chrome://`、`edge://`、Chrome 应用商店页面等，属浏览器限制，与扩展无关。

若你希望缩小范围（例如只匹配若干域名），可把 `manifest.json` 里 `content_scripts[0].matches` 与 `host_permissions` 从 `<all_urls>` 改回具体 URL 模式。

---

## 权限说明

| 权限 | 用途 |
|------|------|
| `storage` | 保存全局配置（`config:v1`）、每页高亮列表、面板位置/折叠状态等。 |
| `tabs` | 从弹窗向当前标签页发送消息（同步开关与颜色等配置）。 |

`host_permissions` 当前为 `<all_urls>`，与全站内容脚本范围一致；上架商店时审核方可能关注广域权限，可按产品需求收窄。

## 项目结构（主要文件）

```
highlight/
├── manifest.json    # MV3 配置、匹配规则、权限
├── content.js       # 划词高亮、存储、精读模式、页内面板、消息处理
├── styles.css       # 页内高亮样式、面板样式、弹窗共用样式
├── popup.html       # 工具栏弹窗界面
├── popup.js         # 弹窗逻辑：读写配置并下发到页面
└── README.md
```
